## 3. FLM 状态机 (Forgetting-Learning-Mastery)
================================================================

### 3.1 词汇池 FLM (src/utils/vocabPool.ts)
四状态机:
  'new' → 'learning' → 'familiar' → 'mastered'

状态转换:
  new       — 首次遇到词汇 (tap event)
  learning  — 需要 L2/L3 深层帮助，或 correctStreak < 2
  familiar  — correctStreak >= 2，interval 倍增
  mastered  — correctStreak >= 4，interval = 30 天

间隔复习 (Spaced Repetition):
  答对: correctStreak++, interval *= 2 (上限 30 天)
  答错: correctStreak = 0, interval = 1 (重置)

VocabPoolEntry 字段:
  wordId, state, interval, lastReview, reviewCount,
  correctStreak, tapCount, maxLevel, lastKpId

### 3.2 技能健康状态机 (src/utils/skillHealth.ts)
四级腐蚀等级:
  'none' (100-76) → 'warning' (75-51) → 'blocked' (50-26) → 'critical' (0-25)

SkillHealthState 字段:
  healthScore (0-100), corruptionLevel, dominantPatternId,
  consecutiveSamePattern, recentErrorCount, totalAttempts,
  blockingPatterns, lastErrorAt, recoveredAt

伤害计算:
  答错: -20 基础伤害 × 严重性权重 (1.0-5.0)
  答对: +8 恢复
  恢复完成: 重置到 100

### 3.3 装备耐久系统 (src/utils/equipment.ts)
  'pristine' (0-7天) → 'worn' (7-21天) → 'damaged' (21-60天) → 'broken' (60+天)

  修复道具:
    锤子 (Hammers): +30 HP
    水晶 (Crystals): +50 HP + 特定加成
    卷轴 (Scrolls): +20 HP + 错误模式免疫

### 3.4 数据存储位置
  localStorage:
    gl_practice_*  — 练习阶段状态 (7 天过期)
    gl_vocab_pool  — 词汇间隔复习状态
    gl_vocab_events — 最近 500 条词汇事件
    gl_guest_profile — 游客档案

  Supabase:
    gl_user_progress.completed_missions (JSONB):
      ._skillHealth[nodeId]     — 技能健康
      ._equipment[missionId]    — 装备耐久
      ._streak_milestones       — 连续登录
      ._char_progression        — 角色成长

## 4. KP → 题目映射
================================================================

### 4.1 映射链路
  KP (294个) → Mission (425个) → Generator (89个) → Question (运行时生成)

### 4.2 Mission 结构 (src/types.ts)
  id: number              — 唯一关卡 ID
  grade: number           — 年级 (7-12)
  unitId: number          — 单元编号
  order: number           — 单元内顺序
  type: QuestionType      — 56 种题型之一
  kpId?: string           — 绑定的 KP ID (如 "kp-1.1-02")
  kn_id?: string          — 规范化知识网络 ID
  data: any               — 生成器专用载荷 (含 generatorType)
  discoverSteps           — 交互式探索序列
  tutorialSteps           — 引导教程
  secret                  — 概念/公式/提示

### 4.3 关卡分布
  Y7:  90 关
  Y8:  78 关
  Y9:  95 关
  Y10: 87 关
  Y11: 60 关
  Y12: 15 关
  总计: 425 关 (404 关已绑定 kpId = 95%)

### 4.4 Variant 概念
- 无显式 variantId — 通过 generatorType + mission.data 参数实现变体
- generatorType (89种) 在运行时根据 DifficultyTier (1-3) 生成不同难度的题目
- 例: QUADRATIC_RANDOM 根据 tier 调整系数范围

### 4.5 考频权重
- 目前无显式考频权重字段
- KP tier ('core'/'both'/'ext') 隐含优先级

## 9. 词汇体系
================================================================

### 9.1 架构
- 无独立 vocab 数据目录 — 词汇系统内嵌于组件
- 三级深度: L1 (定义) → L2 (题意解读) → L3 (方法指导)
- 数据来源: src/utils/mathVocab.ts

### 9.2 词汇与 KP 关系
- 每个词汇通过 lastKpId 关联到最近学习的 KP
- 词汇出现在关卡上下文中 (VocabPopup.tsx)
- 独立于战斗系统的掌握追踪

### 9.3 进度追踪
  客户端 (localStorage):
    gl_vocab_pool   — VocabPoolEntry 数组 (state/interval/streak)
    gl_vocab_events — 最近 500 条 tap 事件

  间隔复习算法: SM-2 变体 (spacedRepetition.ts)
  分析: vocabAnalytics.ts 追踪跨关卡词汇模式

### 9.4 ExamHub 词汇桥接
- Play 通过 ExamHubBridge 组件读取 ExamHub 的 leaderboard 表
- 显示: mastery_pct + mastered_words / total_words
- 教师看板中展示学生词汇进度

# 25Maths Play — 诚实评估与下一步计划

> 写于 2026-03-28，v10.3.0（233关/2389测试/78生成器）
> 这份报告不是给投资人的 pitch，是给自己的诊断书。

---

## 一、产品现在是什么

一句话：**一个工程质量优秀的数学刷题平台，穿了一件三国的外衣。**

### 我们有的

| 能力 | 状态 | 证据 |
|------|------|------|
| 233 道关卡，覆盖 Y7-Y12 | ✅ 已完成 | 6个年级文件，每关≥6步金标准教程 |
| 78 个随机生成器 | ✅ 已完成 | 每次练习题目不同 |
| CIE 0580 课纲映射 | ✅ 内部完成 | 294 个知识点有 kpId 对应 |
| 四阶段递进 (Green→Amber→Red→Battle) | ✅ 已完成 | 从教学→引导→独立→巩固 |
| 教师看板 (7维雷达图+薄弱点分析) | ✅ 已完成 | 可看到每个学生的弱项 |
| 错误诊断 (60种错误模式) | ✅ 已完成 | 能识别"符号错/概念错/计算错" |
| 三币经济 + 军衔系统 | ✅ 已完成 | 游戏化激励闭环 |
| 4 条远征 (桃园/赤壁/蜀道/北伐) | ✅ 已完成 | 关卡制冒险模式 |

### 我们没有的

| 能力 | 状态 | 影响 |
|------|------|------|
| **真实学生使用数据** | ❌ 零 | 不知道产品是否真的有效 |
| **家长可见的进度报告** | ❌ 不存在 | 家长无法知道孩子学了什么 |
| **错误自适应路由** | ⚠️ 框架在，最后一步没接通 | 检测到错误但不会自动调整下一题 |
| **课纲对齐的学生可见性** | ❌ kpId 纯内部 | 学生不知道自己在练哪个考纲知识点 |
| **任何形式的用户分析** | ❌ 零 | 不知道学生在哪里流失、哪里卡壳 |
| **无障碍支持** | ❌ 零 aria 标签 | 约 15% 有特殊需求的学生无法使用 |
| **离线使用** | ❌ 无 PWA/Service Worker | 网络不好的环境无法使用 |
| **内容审核流程** | ⚠️ 只有 bug 报告 | 没有老师/专家审核机制 |

## 二、四个关键人群的真实判断

### 学生："前20分钟有趣，之后在刷题"

**能留住的学生：** 已经有一定基础、喜欢游戏化、老师或家长要求使用的学生。
**留不住的学生：** 基础薄弱、看到 LaTeX 就头疼、没有外部监督的学生。

**核心矛盾：** 6步教程写得再好，学生也会跳过去直接猜答案。Green 阶段的教程是被动阅读，不是主动交互。读教程 < 做错一次看反馈 < 自己发现错误。但产品的主路径是"读教程→答题"，不是"先犯错→再学习"。

### 家长："我看不到孩子学了什么"

家长关心三件事：
1. 孩子今天做了多久？— **看不到**
2. 孩子学了什么知识点？— **看不到**
3. 孩子哪里需要帮助？— **看不到**

没有家长报告 = 家长不会推荐 = 口碑传播链断裂。

### 老师："数据很多，但 so what？"

教师看板有 7 维雷达图、热力图、薄弱点排名。但老师需要的是：

> "张三本周因式分解正确率 40%，全班平均 75%。建议布置 3 道专项练习。"

而不是：

> "张三的 Subject Balance 维度为 0.42/1.0。"

**数据 ≠ 洞察 ≠ 行动。** 看板有数据，缺洞察，更缺行动建议。

### 学校："你有成效证据吗？"

回答是：**没有。**

没有一个班级试点过。没有前后测对比。没有教师反馈。没有学生留存数据。

一个没有被任何真实用户验证过的产品，无论代码写得多好，在学校面前都是零信用。

停止做的事情：

## 四、下一步计划——从"自嗨"到"被验证"

### Phase 1：用户验证（1-2 周）— 最高优先级

**目标：在写任何新代码之前，证明产品对真实学生有效。**

#### 1.1 找 5 个学生做 30 分钟试用

| 学生画像 | 目的 |
|----------|------|
| Y7 基础薄弱（数学 C/D） | 测试教程是否真的能教会 |
| Y7 基础良好（数学 A/B） | 测试是否有挑战性 |
| Y9 备考 IGCSE | 测试课纲覆盖是否足够 |
| Y10 备考 IGCSE | 同上 |
| 1 个英语母语学生 | 测试英文内容质量 |

**操作方式：**
- 坐在学生旁边，全程录屏
- 不做任何引导，只记录他们的困惑
- 记录：在哪停下来了？在哪皱眉了？跳过了什么？问了什么问题？
- 结束后问 3 个问题：
  1. "你觉得你学到了什么？"
  2. "哪个地方让你困惑？"
  3. "你会自己再来用吗？为什么？"

#### 1.2 找 2 个老师做 15 分钟看板演示

- 给老师看教师看板（用模拟数据）
- 问："看到这些数据，你下一步会做什么？"
- 如果老师说不出具体行动 → 看板设计失败，需要重做

#### 1.3 找 2 个家长做 5 分钟展示

- 给家长看学生端界面（而不是看板）
- 问："你觉得你的孩子在学习还是在玩游戏？"
- 问："你想看到什么样的进度报告？"

**Phase 1 的产出：**
- 一份用户访谈记录（原始笔记，不美化）
- 一份"学生真正卡壳的地方"清单
- 一份"老师/家长真正想看的东西"清单

---

### Phase 2：基于 Phase 1 反馈的最小修复（2-4 周）

**只修 Phase 1 发现的问题。不做任何"我觉得应该做的"功能。**

以下是基于现有代码审计的**预判**（Phase 1 可能会推翻这些）：

#### 2.1 家长周报（预计 Phase 1 会验证需求）

一页简洁报告，每周自动生成：

```
本周学习报告 — 张三 (Y9)
━━━━━━━━━━━━━━━━━━━━━━━
练习时长：45 分钟（3 次登录）
完成关卡：8 关（因式分解×3、方程×2、概率×3）
正确率：72%（上周 65% ↑）
薄弱点：符号运算（建议在家练习正负号规则）
老师评语：[教师可选填]
```

技术上可行：数据已在 Supabase (`gl_user_progress`, `gl_battle_results`)。缺的是聚合+展示层。

#### 2.2 错误自适应路由——接通最后一步

现状：错误模式检测 ✅ → 补救路径构建 ✅ → 自动路由 ❌

需要做的：
- 当同一错误模式出现 3 次 → 自动插入 1 道补救题（不是建议，是强制）
- 补救题来自已有的 Recovery Pack 系统
- 学生看到的是："军师紧急来报——先解决这个问题再继续前进！"（融入叙事）

#### 2.3 教师看板加行动建议

在现有薄弱点面板下方，加一行具体建议：

> "建议：张三的因式分解正确率（42%）显著低于班级平均（78%）。
> → [一键布置] 因式分解专项 5 题"

技术上可行：已有 `computeAlerts()` 和 assignment 系统。缺的是建议文案生成逻辑。

#### 2.4 课纲可见性

在 Practice/Battle 模式的题目上方，加一行小字：

> `CIE 0580 · 2.1 Algebra — Factorising`

让学生和老师都知道这道题对应哪个考纲知识点。数据已有（kpId → kp-registry），只需渲染。

### Phase 3：学校试点（4-8 周）

**目标：1 所学校，1 个班级，4 周试点，收集成效数据。**

#### 3.1 试点设计

| 要素 | 计划 |
|------|------|
| 学校 | 找 1 所有 IGCSE 课程的学校（国际学校或双语学校） |
| 班级 | 1 个 Y9 或 Y10 班级（20-30人） |
| 时长 | 4 周，每周 2 次，每次 20 分钟 |
| 对照 | 同年级另一个班级不使用（如果学校允许） |
| 测量 | 前测（第 0 周）+ 后测（第 4 周），用相同的 10 道标准题 |
| 数据 | 登录频率、完成率、正确率变化、错误模式分布 |

#### 3.2 试点前的技术准备

| 必须项 | 说明 |
|--------|------|
| 基础 analytics | 至少记录：每次登录时间、每题停留时间、每题正误 → Supabase |
| 稳定性 | 确保 30 人同时使用不崩溃 |
| 教师培训 | 15 分钟教会老师使用看板 |
| 家长通知 | 一页纸说明"这是什么，你的孩子会做什么" |

#### 3.3 试点后的产出

- 前后测成绩对比数据
- 学生使用行为数据（哪些关卡最受欢迎、哪些流失率最高）
- 教师反馈（看板有用吗？内容对齐课堂吗？）
- 学生反馈（你会推荐给朋友吗？为什么？）

**这份数据是产品走进下一所学校的敲门砖。没有这份数据，一切都是空谈。**

---

### Phase 4：根据试点数据迭代（第 8 周+）

Phase 4 现在不规划。因为 Phase 3 的数据可能会完全推翻当前的产品假设。

可能的方向：
- 如果学生说"教程太长了" → 重新设计为交互式 2-3 步
- 如果老师说"和我的教学进度不匹配" → 加入自定义关卡顺序
- 如果数据显示"学生只做 Green 不做 Red" → 重新设计阶段门槛
- 如果数据显示"Y7 用得多，Y10 没人用" → 聚焦低年级内容

**不预判。用数据说话。**

## 七、给自己的话

你建了一座漂亮的数学城堡。233 间房间，每间都有精心设计的 6 步教程，三国武将在走廊里巡逻，78 台随机出题机器日夜运转。

但城堡里一个学生都没有。

城堡不需要更多房间了。它需要的是：**打开门，让第一个学生走进来，然后看看他迷路了没有。**

如果他没有迷路——恭喜，你做了一个好产品。
如果他迷路了——那才是真正的开发计划。

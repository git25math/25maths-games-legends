# 教师看板开发计划 v2

> 基于教师真实工作流设计，按使用频率排序。
> 写于 2026-03-28，v10.4.0
> 取代旧版 Phase A-D 计划（已完成）

---

## 设计原则

1. **30 秒原则**：老师打开看板，30 秒内得到今天需要知道的一切
2. **一键原则**：布置作业、分享链接、生成报告——都应该是一键操作
3. **不加工作量**：每个功能必须减少老师的工作量，不是增加

## 新开发计划

### Phase D1：作业闭环（第 1 周）

> **场景**：老师布置完作业，5 秒内让全班知道。

| # | 功能 | 说明 | 工作量 |
|---|------|------|--------|
| D1.1 | **作业分享链接** | 老师点"复制链接" → 得到 `play.25maths.com?hw=xxx` → 发微信群 → 学生点开自动跳到作业列表 | 小 |
| D1.2 | **地图作业角标** | 被布置的关卡在 MapScreen 显示 📝 徽标，学生一眼看到哪些关卡是作业 | 小 |
| D1.3 | **截止倒计时** | 学生端作业卡片显示"还剩 2 天"而不是 "2026-04-05"（人类可读） | 小 |

**验收标准**：老师创建作业 → 复制链接 → 发群 → 学生点开 → 看到作业 → 完成关卡 → 老师看到完成率。全程 < 2 分钟。

---

### Phase D2：今日摘要（第 2 周）

> **场景**：周一早上 7:50，老师打开看板，30 秒内知道本周重点。

| # | 功能 | 说明 | 工作量 |
|---|------|------|--------|
| D2.1 | **今日摘要卡片** | 看板最顶部一张卡：昨天活跃 X 人 / 作业完成 Y% / Z 人需关注 / 班级本周进步率 | 小 |
| D2.2 | **按知识点查看弱生** | 点击 KPWeaknessPanel 的某个 KP → 展开该 KP 弱的学生名单 + 各自正确率 | 中 |
| D2.3 | **一键布置补习** | KP 行动建议变成按钮：点击 → 自动选择该 KP 关联的 5 道关卡 → 创建作业 → 完成 | 中 |

**验收标准**：老师看到"因式分解 38%，8 人薄弱" → 点击 → 看到 8 个学生名字 → 点"一键补习" → 作业自动创建。全程 < 15 秒。

---

### Phase D3：报告系统（第 3 周）

> **场景**：周五给年级组长交周报，月底给家长会准备材料。

| # | 功能 | 说明 | 工作量 |
|---|------|------|--------|
| D3.1 | **班级周报** | 一键生成 → 一页 HTML：本周活跃人数/答题量/正确率趋势/进步最大 3 人/最需关注 3 人 → 可复制纯文本或打印 | 中 |
| D3.2 | **学生月报（给家长）** | 点学生 → "生成家长报告" → 一页：本月做了 X 题/正确率 Y%/掌握 Z 个知识点/薄弱点建议 → 可复制发微信 | 中 |
| D3.3 | **8 周趋势图** | WeeklyTrend 升级为 8 周折线图：班级平均正确率 + 活跃人数 + 平均进度 | 中 |

**验收标准**：老师点"生成周报" → 3 秒内看到完整报告 → 复制文本 → 粘贴到微信发给组长。

---

### Phase D4：班级管理增强（第 4 周）

> **场景**：新学期开学，老师要建 3 个班，每班 30 人。

| # | 功能 | 说明 | 工作量 |
|---|------|------|--------|
| D4.1 | **动态班级筛选** | 老师创建的班级自动出现在顶部筛选芯片中（替代硬编码 CLASS_GROUPS） | 中 |
| D4.2 | **班级成员管理** | 查看某班全部成员 → 移除学生 → 查看空位 | 中 |
| D4.3 | **批量导入学生** | 老师粘贴学生名单（一行一个名字）→ 批量创建账号 + 自动加入班级 | 大 |

**验收标准**：老师创建班级 → 发邀请码 → 30 个学生加入 → 老师在看板里按班级筛选看到所有人。

---

### Phase D5：智能建议（第 5-6 周）

> **场景**：老师不知道这周该教什么，看板告诉他。

| # | 功能 | 说明 | 工作量 |
|---|------|------|--------|
| D5.1 | **本周 3 条建议** | 基于全班数据自动生成："① 重点复习因式分解（12人弱）""② 张三 5 天未登录""③ 李四进步快可挑战" | 中 |
| D5.2 | **智能作业** | 点"自动布置" → 系统根据全班薄弱点选择 5-10 道关卡 → 预览 → 确认创建 | 大 |
| D5.3 | **考试预测** | 基于 KP 掌握率预估 CIE Paper 2（Core）/ Paper 4（Extended）得分区间 | 大 |

## 不做的事情

| 不做 | 原因 |
|------|------|
| 站内聊天/消息 | 老师有微信群，用链接分享更轻量 |
| Google Classroom 集成 | 先验证独立价值 |
| 家长独立登录 | 用分享链接 + 报告代替 |
| 视频批改 | 超出当前产品范围 |

| 场景 | 目标时间 |
|------|---------|
| 老师创建班级 → 学生全部加入 | < 2 分钟 |
| 老师布置作业 → 学生看到 | < 1 分钟 |
| 老师打开看板 → 获取今日摘要 | < 30 秒 |
| 生成班级周报 | 1 键，< 5 秒 |
| 生成家长月报 | 1 键，< 5 秒 |
| 发现全班薄弱点 → 布置补习作业 | < 15 秒 |

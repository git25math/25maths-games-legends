# 25Maths Play — 故事化数学闯关学习平台 开发计划

> **项目地址**：https://play.25maths.com
> **仓库**：git25math/25maths-games-legends
> **版本**：v1.0.0 | 最后更新：2026-03-17

## Context

**问题**：传统数学学习枯燥、抽象、脱离情境。学生刷题时缺乏动力，知识点之间缺乏联系感。

**解决方案**：将 CIE 0580 的 294 个知识点包装成三国演义故事线的闯关游戏。每个知识点 = 一个战役情境，学生在"打仗"的过程中自然掌握数学。

**核心信念**：学生不是不喜欢数学，是不喜欢"无故事的数学"。当诸葛亮教你用抛物线计算箭矢轨迹时，二次函数突然有了意义。

**已完成基础**：
- ✅ Firebase→Supabase 迁移，ExamHub 账号共享
- ✅ 模块化代码架构（25 个源文件，215 行 App.tsx）
- ✅ Green/Amber/Red 三级难度系统
- ✅ 49 个手写关卡 + tutorialSteps
- ✅ 294 KP 元结构 + 三国故事主题 + 哈罗课程映射
- ✅ 部署到 play.25maths.com

## 一、为什么这个方案能让学生爱上数学？

### 1.1 教育心理学基础

| 原理 | 传统做法 | Play 做法 | 效果 |
|------|---------|----------|------|
| **情境学习** (Situated Learning) | "解方程 3x=15" | "三把长剑 15 金，求单价" | 知识有了用武之地，不再抽象 |
| **即时反馈** (Immediate Feedback) | 做完一整张试卷才知道对错 | 每一步即时验证，错了鼓励重来 | 缩短反馈回路，加速学习 |
| **渐进挑战** (Flow Theory) | 所有学生做一样难度的题 | Green→Amber→Red 自适应难度 | 保持心流状态，不焦虑不无聊 |
| **叙事驱动** (Narrative Engagement) | "第一题、第二题、第三题" | "草船借箭→火烧赤壁→华容道" | 故事推进感让学生想知道下一关 |
| **角色认同** (Identity) | 学生 = 做题机器 | 学生 = 诸葛亮/曹操/周瑜 | 代入感让数学变成"我的战斗" |
| **前置依赖** (Prerequisite Chain) | 跳着做题，基础不牢 | 必须先过勾股定理才能解锁三角函数 | DAG 保证知识递进 |
| **三位一体** | 看书→做题（两步） | 视频讲解→闯关实战→真题练习 | 理解→应用→巩固 完整闭环 |

### 1.2 学生体验设计

**一个 Y10 学生的典型 30 分钟**：

```
1. 登录 play.25maths.com → 选择曹操为武将
2. 进入 "赤壁之战·代数篇" 地图
3. 看到 "火攻轨迹" 章节，点击 kp-2.10-01 "草船借箭"
4. 选择 Green (名师指路) 难度
5. 📜 锦囊妙计弹出：二次函数顶点公式
6. 🎮 闯关开始：
   - 周瑜："大雾弥漫，箭矢轨迹 h = -x² + 10x，求最高点！"
   - Step 1: 诸葛亮引导 → "顶点 x = -b/(2a)"
   - Step 2: 学生填入 x = 5 ✓
   - Step 3: "最高 25 丈！妙计成功！" 🎉
7. 获得功勋值 300，Green 完成标记亮起
8. 🎬 点击"看视频讲解" → manim 视频 kp-2.10-01 (4 分钟)
9. 📝 点击"练真题" → ExamHub 调取 kp-2.10-01 相关真题
10. 回到地图 → Amber 难度已解锁 → 继续挑战
```

**关键体验指标**：
- **5 分钟内上手**：选武将→选年级→开始闯关
- **每关 3-5 分钟**：短平快，适合课间/通勤
- **失败不惩罚**：HP 耗尽 → 鼓励语 → 重来，不清进度
- **成就可见**：Green/Amber/Red 三色圆点像勋章一样积累
- **社交分享**：多人对战模式，同学之间 PK

### 3.2 故事推进感设计原则

**不是"用三国包装数学题"，而是"用数学推动三国剧情"**：

```
❌ 错误示范：
   "曹操要你算 3x = 15，求 x"
   → 学生感受：还是做题，只是换了个名字

✅ 正确示范：
   "刘关张桃园结义，要买三把宝剑。铁匠说三把一共 15 金。
    张飞急了：'老子不识数，亮哥你来算！每把多少钱？'
    诸葛亮微微一笑：'三把 15 金，即 3x = 15，两边除以 3...'
    你填入 x = 5。
    张飞大喜：'5 金一把，实惠！俺老张买了！'
    🎉 结义成功！三兄弟举杯共饮。"
   → 学生感受：我帮张飞算出了剑价，我是故事的一部分
```

**叙事渐进规则**：
1. **前关结果影响后关叙述** — "上一关你帮关羽算出了云梯长度 10 丈，现在他正扛着这架梯子冲向城墙..."
2. **每 topic 有一个小高潮** — 最后一个 KP 是战役的决定性时刻
3. **每 chapter 有一个大结局** — 完成全部 KP 后解锁章节总结动画
4. **人物成长与数学同步** — 学完 CH1 的刘备从"织席贩履"变成"一方诸侯"

### 3.4 单个 KP 的完整打磨流程

以 **kp-2.5-01 一元一次方程** 为例：

```
Step 1: 故事创作 (Story)
  ├── 读三国原典相关情节
  ├── 设计情境："张飞买剑 3x=15"
  ├── 写叙事者台词（中英双语）
  └── 确保故事自然引出数学，不是硬塞

Step 2: 教学设计 (Tutorial)
  ├── Green 模式 3-5 步教学
  │   ├── Step 1: 诸葛亮介绍情境 + 列方程
  │   ├── Step 2: 引导操作（两边除以 3）
  │   └── Step 3: 验证答案 + 故事结局
  ├── Amber 模式: 给公式提示，不给步骤
  └── Red 模式: 纯题目 + 故事背景

Step 3: 题目数据 (Question Data)
  ├── 设计 3-5 组随机参数（不是每次同一道题）
  ├── 确定输入字段类型和验证逻辑
  └── 设计视觉展示（坐标图/三角形/方程式等）

Step 4: 视频对接 (Video Link)
  ├── 确认 math-video-engine 中的 kp-2.5-01.yaml 存在
  ├── 视频完成后在闯关页面加 "🎬 看视频" 按钮
  └── 视频内容与闯关故事一致

Step 5: 练习对接 (ExamHub Link)
  ├── 从 question-kp-map-cie.json 调取该 KP 的真题
  ├── 在闯关完成后推荐 "📝 练真题"
  └── 练习结果回写 ExamHub 的 vocab_progress
```

## 六、风险分析与应对

### 6.1 已识别风险

| 风险 | 级别 | 影响 | 应对策略 |
|------|------|------|---------|
| **游戏机制单一** | 🔴 高 | 当前只有"填数字"一种交互，294 KP 中 T1(概念分类)、T5(图像)、T6(几何推理) 需要不同交互 | Phase 2 新增 4 种题型组件：多选题(T1)、排序题(T2)、图像匹配(T5)、角度标注(T6) |
| **故事覆盖粒度不够** | 🟡 中 | three-kingdoms.ts 只定义到 topic 级，294 个 KP 级故事需要逐个创作 | P0 的 32 个 KP 手工写故事；其余用 Gemini 按模板批量生成 + 人工校对 |
| **49→294 关卡迁移** | 🟡 中 | 现有 49 个手写关卡与新 KP 体系 ID 不兼容 | Phase 2 保留旧关卡作为 fallback，新系统并行运行，渐进替换 |
| **视频依赖** | 🟡 中 | "三位一体"依赖 manim 视频就绪，但 272 个视频尚未全部完成 | 视频未就绪的 KP 隐藏"看视频"按钮，不影响闯关和练习 |
| **哈罗课程映射准确性** | 🟡 中 | harrow.ts 中的 Y7-Y11 topic 分配基于通用 CIE 进度，可能与哈罗实际教学计划有偏差 | 需要与哈罗数学组确认，支持教师自定义年级 KP 映射 |
| **故事分散注意力** | 🟢 低 | 部分学生可能觉得故事多余，只想做题 | Red 模式本身就是纯题目无故事；添加"极简模式"开关 |

### 6.2 当前 checkCorrectness 覆盖缺口

现有验证逻辑只覆盖 24 种 question_type，但 294 KP 的 8 种 script_type 需要不同的游戏机制：

| Script Type | 当前支持 | 需要新增 | 示例 |

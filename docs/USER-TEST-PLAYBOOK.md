## 三、测试流程（每人 30 分钟）

### 开场（2 分钟）

说这些：

> "我在做一个数学学习的网站，想看看你用起来感觉怎么样。
> 这不是考试，没有对错。我想测试的是这个网站，不是你。
> 你可以随便点，随便试。遇到不懂的地方就说出来。
> 我不会帮你，因为我想看你自己能不能搞明白。
> 可以随时停下来。"

### 自由探索（20 分钟）

**你要做的：闭嘴。**

- 不解释
- 不引导
- 不说"试试那个按钮"
- 不在学生犯错时纠正

**你要记录的：**（用记录表，见第四节）

每隔 1-2 分钟记一条：
- 学生在做什么（"在读教程第3步"/"在输入答案"/"在看地图"）
- 学生的表情/反应（"皱眉"/"笑了"/"叹气"/"加快点击速度"）
- 学生说了什么（"这是什么意思？"/"哦我懂了"/"好无聊"）

**特别注意这些信号：**
- 🔴 **停下来超过 10 秒** → 记录"卡在哪"
- 🔴 **点了跳过教程** → 记录"跳过了哪个关卡的教程"
- 🔴 **连续做错 3 题** → 记录"是否触发修复拦截""学生选了什么"
- 🟡 **快速点击不看内容** → 记录"在刷而不是学"
- 🟢 **回头重看教程** → 记录"主动复习"
- 🟢 **做对时明显开心** → 记录"正向反馈有效"

### 三个问题（5 分钟）

测试结束后，问这三个问题。**逐字记录学生的原话，不要改写。**

**问题 1：** "你觉得你刚才学到了什么？"

> （如果学生说"不知道"或"没有"——这是最重要的数据点。）

**问题 2：** "哪个地方让你最困惑？"

> （追问："是看不懂文字？还是不知道要做什么？还是数学本身不会？"）

**问题 3：** "如果没有人要求你用，你会自己再来用吗？为什么？"

> （这个问题的答案决定产品是否有自然留存力。）

### 收尾（3 分钟）

- 感谢学生
- 问学生是否愿意两周后再来试一次（为 Phase 3 试点做准备）
- 关闭录屏

### 5.2 看 Supabase 数据

测试后登录 Supabase Dashboard（项目 ref: `jjjigohjvmyewasmmmyf`），查询：

```sql
-- 每个学生的答题数和正确率
SELECT user_id, count(*) as attempts,
       round(avg(case when is_correct then 1 else 0 end)::numeric, 2) as accuracy,
       round(avg(extract(epoch from age(created_at, lag(created_at) over (partition by user_id order by created_at))))::numeric, 1) as avg_gap_secs
FROM user_attempt_log
WHERE created_at > now() - interval '7 days'
GROUP BY user_id;

-- 最常见的错误类型
SELECT primary_error_pattern_id, count(*) as cnt
FROM user_attempt_log
WHERE NOT is_correct AND created_at > now() - interval '7 days'
GROUP BY primary_error_pattern_id
ORDER BY cnt DESC
LIMIT 10;

-- 平均用时最长的题目（学生卡壳的地方）
SELECT question_id, round(avg(extract(epoch from age(created_at, lag(created_at) over (partition by user_id order by created_at))))::numeric, 1) as avg_secs, count(*) as attempts
FROM user_attempt_log
WHERE created_at > now() - interval '7 days'
GROUP BY question_id
HAVING count(*) >= 3
ORDER BY avg_secs DESC
LIMIT 10;

-- 教程跳过率
SELECT count(*) FILTER (WHERE question_id LIKE '%-skip-tutorial') as skips,
       count(*) as total
FROM user_attempt_log
WHERE created_at > now() - interval '7 days';
```

### 5.3 做决策

拿着观察记录 + Supabase 数据，回答三个问题：

1. **学生真正的痛点是什么？**（观察到的，不是你猜的）
2. **产品的哪个部分最有价值？**（学生主动使用的，不是你觉得好的）
3. **下一步修什么？**（只修测试中发现的问题，不修你想到的功能）

**把答案写进 `docs/USER-TEST-FINDINGS.md`。这份文档将取代 DEVELOPMENT-PLAN.md 成为真正的开发指南。**

## 六、时间线

```
本周：联系 3-5 个学生，约时间
下周：执行 3-5 次 30 分钟测试
下周末：分析数据，写 USER-TEST-FINDINGS.md
再下周：根据发现，开始 Phase 2 修复
```

## 七、提醒自己

测试时你会很想帮学生。忍住。

学生卡壳时你会很想解释。忍住。

学生说"好无聊"时你会很难受。记下来。

**你不是在展示你的作品。你是在发现它的缺陷。**

每一个困惑、每一次犹豫、每一句"这是什么意思"——都是比 100 个新生成器更珍贵的数据。

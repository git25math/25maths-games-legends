# 三角洲行动（Delta Force）游戏设计调研 × 25Maths Play 借鉴分析

> **调研日期**: 2026-03-20
> **调研背景**: 学生群体中三角洲行动热度极高，日活峰值超 1200 万，注册用户 8000 万+。本文档分析其成功的设计机制，并提出可落地到 25Maths Play 教育游戏的借鉴方案。

---

## 一、为什么学生喜欢三角洲行动？

### 1.1 「风险-回报」的心跳感

烽火地带模式的核心循环是「带入→搜刮→战斗→撤离→结算」。玩家带着自己的装备进场，搜到的战利品都是"真金白银"——但一旦阵亡全部丢失。这种 **"得失感真实化"** 产生的肾上腺素，是普通对战游戏给不了的。

设计者建立了"参与战斗的收益是避战策略的 3-5 倍"的物质激励模型，将战斗行为与赚钱目标绑定，催生积极的游戏风格。

### 1.2 「成为自己的英雄」的自主感

GDC 2025 演讲揭示其核心设计哲学：不强制特定玩法风格。

- **四象限能力框架**：基于进攻/防守、主动/被动两个维度划分角色风格
- 每个干员通过调配 6 大技巧维度实现差异化体验
- "枪手"和"策略家"都能找到归属
- **玩家感觉自己在做选择，而不是被系统推着走**

### 1.3 精心调校的心流区间

- TTK（击杀时间）从 200ms 提升到 350ms，给予容错空间让新手也能"打出操作"
- 二次救援机制保证"先手不等于必胜"
- **永远让玩家觉得"差一点就赢了"，而不是"毫无希望"**

**技能设计三支柱**:
1. "技能不取代枪法，只创造战术机会"
2. 所有技能遵循"风险-收益"准则，提供精通空间
3. 为防守方预留"清晰、直觉的反制空间"

### 1.4 多模式防倦怠

三种截然不同的模式覆盖不同偏好：

| 模式 | 玩法 | 对应玩家偏好 |
|------|------|-------------|
| 烽火地带 (Hazard Operations) | 搜打撤 PvPvE | 策略型 / 紧张感 |
| 全面战场 (Havoc Warfare) | 32v32 大战场 + 载具 | 快节奏爽快感 |
| 黑鹰坠落 (Black Hawk Down) | 16 关 PvE 合作战役 | 故事驱动 / 合作 |

加上定期轮换的限时模式（团队竞技、空中优势等），**玩家永远有新鲜感，不会困在单一循环里**。

### 1.5 赛季通行证的长线牵引

- 每赛季带来新干员、新地图、新外观奖励
- 每日任务（难度适中）+ 每周挑战（更高难度）+ 赛季里程碑
- 通行证进度条让玩家感到"每天玩一点都有意义"
- **碎片化时间也能产生成就感**

### 1.6 社交绑定

- 4 人小队组队 + 队内语音 + 标记系统
- 倒地救助机制强化队友依赖
- 好友系统 + UID 添加
- **朋友之间互相拉回来玩的社交黏性，是留存的终极武器**

## 二、核心设计机制详解

### 2.1 烽火地带五环闭合循环

```
带入 → 搜刮 → 战斗 → 撤离 → 结算
 ↑                                ↓
 └──── 资源再投入 / 升级装备 ←────┘
```

- **带入阶段**: 选择装备，受成本/价值上限约束（确保公平）
- **搜刮阶段**: 战利品池 + 刷新逻辑 + 位置权重 + 稀有度梯度
- **战斗阶段**: 可选交战或避战，包容多种游玩风格
- **撤离阶段**: 限时找到撤离点并成功撤退
- **结算阶段**: 根据撤离状态及携带物品进行奖励或惩罚

### 2.2 物品经济系统

- **多层货币**: 基础货币 + 高级货币/信用点 + 特定材料
- **市场系统**: 玩家驱动的拍卖行与商店交易（税制、手续费、定价）
- **装备系统**: 庞大装备库含性能参数与改装槽位深度
- **损耗机制**: 使用降低耐久，需货币 + 材料维修，形成资源沉没点
- **宏观调控**: 通过 NPC 回收、生产消耗、稀缺资源控制避免通胀

### 2.3 进度与留存系统

- **7 级段位系统**: 击杀/助攻/目标/胜利均贡献，跨模式共享
- **跨平台存档**: PC/Mobile/Console 进度互通，不重复劳动
- **新手保护**: 低成本入场（AI 角色 ScavRun）、隐蔽软 MMR 匹配、等级保护
- **渐进信息解锁**: 地图情报（撤离点、危险区域）随等级开放

### 2.4 情绪设计弧线

> 系统性塑造冲突，让玩家的旅程充满挑战；随着游戏进程推进，困难越来越大，
> 带来持续提升的情绪张力；当玩家"冲出重围成功撤离"时，所有积累的情绪在此刻释放，
> 转化为深层成就感。这不仅是创造高光时刻，而是"创造属于玩家自己的英雄成长史诗"。

## 三、机制映射：从射击游戏到教育游戏

| 三角洲行动机制 | 设计本质 | 教育游戏可借鉴的抽象 |
|---|---|---|
| 搜打撤循环 | 高风险高回报的「得失感」 | 学习中的「押注-挑战-收获」循环 |
| 赛季通行证 | 长线目标 + 短线任务的双轨进度 | 学期目标 + 每日练习的双轨成长 |
| 多模式 | 防倦怠 + 适配不同偏好 | 多种学习模式（教程/练习/挑战/对战） |
| 干员系统 | 个性化能力 + 自主选择 | 角色成长路径 + 技能树 |
| TTK 容错 | 降低挫败感、保持心流 | 自适应难度、错误宽容机制 |
| 小队合作 | 社交黏性 + 互助 | 组队学习、班级挑战 |
| 装备损耗/维修 | 资源管理的沉浸感 | 知识「遗忘曲线」的具象化 |
| 沙盒竞技 | 工具化技能 + 创意组合 | 多种解题策略的鼓励 |
| ScavRun 模式 | 零成本试错入门 | 无惩罚的自由探索练习区 |

## 四、25Maths Play 具体借鉴方案

### 4.1 高优先级（v6.x 可做）

#### A.「远征模式」— 搜打撤的教育化改编

**概念**: 学生带着"粮草"（虚拟资源）进入一张知识地图，每个节点是一道题。答对获得战利品（经验/装备），答错消耗粮草。最终需要成功"撤离"（完成足够题目）才能保留收获。

**核心循环**:
```
选择装备(已掌握的知识点作为buff)
    → 进入地图(随机题目节点)
    → 解题搜刮(答对得战利品)
    → 遭遇战(限时高难度题，风险更高收益更大)
    → 撤离结算(达到目标分数即可撤离保留全部收获)
```

**关键设计**:
- 粮草不足时可以选择"提前撤离"保留已有收获，或者冒险继续
- 借鉴三角洲"参与战斗收益是避战的 3-5 倍"原则 → 做难题的回报远大于做简单题
- FLM mastered 的知识点作为"装备 buff"（如"代数精通"给代数题额外得分）

**与现有系统整合**:
- 基于 `generateMission.ts` 的随机题目生成
- FLM 4 态作为"装备等级"的数据源
- `BATTLE_TIMING` 控制节奏

#### B. 赛季通行证 → 「学期成长手册」

**概念**: 每学期一个通行证，内含分层任务和奖励。

**任务层级**:
| 层级 | 频率 | 示例 | 奖励 |
|------|------|------|------|
| 每日 | 日更 | 完成 3 道练习题 | 5 功勋 + 手册经验 |
| 每周 | 周更 | 完成 1 个单元的 Green→Red | 30 功勋 + 外观碎片 |
| 里程碑 | 赛季 | 累计掌握 20 个知识点 | 新角色皮肤/称号/边框 |

**进度可视化**:
- 通行证进度条（类似三角洲赛季手册）
- 免费轨道 + 高级轨道（教师可为班级开通）

**与现有系统整合**:
- Supabase `student_profile` 存储通行证进度
- 新表 `season_pass_progress`
- 前端新增 `SeasonPassScreen.tsx`

#### C.「容错机制」升级 — 差一点就对了

**概念**: 答题不再是非黑即白。

**借鉴三角洲 TTK 容错设计**: 让玩家永远觉得"再试一次就能赢"。

**实现方案**:
| 情况 | 当前行为 | 改进后 |
|------|---------|--------|
| 答案接近正确（如 x=4 答成 x=4.0） | 判错 | 判对 + 精确提示 |
| 方法正确最后一步算错 | 判错 | 部分得分(50%) + "方法正确，计算有误" |
| 完全错误 | 判错 | 判错 + 展示第一步提示 |

**与现有系统整合**: 扩展 `checkCorrectness.ts` 增加 `partialCredit` 返回值。

#### D.「班级远征」— 社交组队

**概念**: 3-4 人组队挑战一组关卡。

**借鉴三角洲小队系统 + 倒地救助**:
- 每人负责不同知识点的题目
- 队友可以"救助"一次（给被困队员一个提示）
- 全队通关才算成功
- 实时显示队友进度

**与现有系统整合**: 已有 Supabase 实时功能 + `useMultiplayer` hook + `LobbyScreen`。

### 4.2 中优先级（v7.x 规划）

#### E. 角色技能树 → 「武将修炼」

每个三国武将有独特技能树：

```
诸葛亮 (策略型)
├── 锦囊妙计: 每日获得 1 次免费提示
├── 运筹帷幄: 策略题额外 +20% 得分
└── 七星灯续: 答错时 10% 概率不扣血

关羽 (速度型)
├── 千里走单骑: 限时题延长 3 秒
├── 温酒斩华雄: 10 秒内答对额外 +50% 得分
└── 义薄云天: 组队时队友得分 +10%
```

**与现有系统整合**: 扩展 `src/data/characters.ts` 的 Character 类型，新增 `skillTree` 字段。

#### F.「知识遗忘」可视化

把 FLM 的 mastered 衰退机制包装成"装备耐久度"概念：

- 每个已掌握知识点显示为一件"装备"
- 7 天不复习 → 耐久度下降 → 视觉变暗/出现裂纹
- 14 天 → "损坏"状态 → 需要"维修"（做 3 道复习题）
- 30 天 → 掉回 uncertain 状态

**借鉴**: 三角洲的装备损耗/维修系统，让抽象的遗忘曲线变成可感知的游戏元素。

**与现有系统整合**: 基于 `REFRESH_INTERVALS [7,14,30]` 天的现有衰退复查机制。

#### G. 多模式防倦怠

| 新模式 | 类型 | 三角洲对标 | 描述 |
|--------|------|-----------|------|
| 速算竞赛 | 限时对战 | 团队竞技 | 30 秒内尽可能多答对，PvP 排名 |
| 知识探索 | 开放地图 | 烽火地带 | 自由选择路径的知识地图 |
| Boss 战 | 综合挑战 | 黑鹰坠落 | 多知识点综合题，叙事驱动 |
| 每日试炼 | 轮换 | 限时模式 | 每天不同规则的特殊挑战 |

### 4.3 长期愿景（v8+）

#### H. 学习经济系统

- 虚拟货币"功勋"的用途扩展（购买外观、解锁关卡、兑换提示）
- 简单的"资源管理"（每日精力值限制防沉迷，同时制造稀缺感）
- 教师可以"悬赏"特定任务（如"本周掌握分数加减的同学获得 xxx 奖励"）

## 五、三条核心设计原则

三角洲行动最值得学的不是具体机制，而是 3 个设计原则：

### 原则 1: 「参与战斗的收益是避战的 3-5 倍」
→ **让做难题的回报远大于做简单题。** 目前 25Maths 的 Red 模式分数倍率只有 2x，可以进一步拉大差距，让学生有动力挑战更难的内容。

### 原则 2: 「技能不取代枪法，只创造战术机会」
→ **辅助工具不替代思考，只降低门槛。** 提示系统不应该直接给答案，而应该给"思路的起点"。计算器辅助计算，但不辅助思考。

### 原则 3: 「创造属于玩家自己的英雄成长史诗」
→ **每个学生都有自己的学习轨迹和成就故事。** 不只是排行榜上的一个数字，而是"我从完全不懂分数，到三星通关分数单元"的个人叙事。
